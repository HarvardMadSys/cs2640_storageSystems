CS2640 Modern Storage Systems
Course Project

Project Overview

The course project is an important component of this class, allowing students to apply the concepts learned in lectures to real-world distributed systems problems. Students will work in teams to design, implement, and evaluate a distributed system.

Project Weight: 40% of final grade
Team Size: 2-3 students
Duration: Throughout the semester

Project Timeline

Milestone Due Date Weight Deliverables
Team Formation Week 2 - Submit team members via form
Project Proposal Week 4 10% 2-page proposal document
Design Document Week 7 15% System design and architecture
Milestone Check-in Week 10 10% Progress presentation
Final Submission Week 14 45% Code, report, demo
Final Presentation Week 15 20% 15-minute presentation

Project Ideas

Below are some suggested project topics. Students are also welcome to propose their own ideas.

Distributed Key-Value Store

Build a fault-tolerant distributed key-value store with replication and consistency guarantees.

Distributed File System

Implement a distributed file system with support for concurrent access and fault tolerance.

Consensus Protocol Implementation

Implement and evaluate a consensus protocol such as Raft or Paxos.

Distributed Machine Learning

Build a system for distributed training of machine learning models.

Microservices Architecture

Design and implement a microservices-based application with service discovery and load balancing.

Requirements

Technical Requirements

  • The system must be distributed across at least 3 nodes
  • Must handle node failures gracefully
  • Must include comprehensive testing
  • Code must be well-documented

Report Requirements

  • System architecture and design decisions
  • Implementation details
  • Evaluation methodology and results
  • Discussion of limitations and future work

Presentation Requirements

  • 15-minute presentation with Q&A
  • Live demo of the system
  • All team members must participate
